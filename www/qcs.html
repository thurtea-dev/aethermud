Quick Creation System

Staff of Creation verbs for rooms, NPCs, weapons, and armor. Work in your realm sandbox first.

Core verbs

create room <dir> <filename>
create npc|weapon|armor <filename>
modify here|<filename> <property> <value>
add exit <dir> <path>
add <npcfile> to here
add <npcfile> to <roomfile>
delete <object>
about <filename|here>
more <filename|here>
clone <filename>
purge <object>

Properties

Room:   short, long, light, indoors
NPC:    name, short, long, level, hp, race, occ, align, mdc
Weapon: name, short, long, wc, type, damage (NdN), mdc (0/1)
Armor:  name, short, long, ac, ar, sdc, mdc, slot

Enemy durability: HP, SDC, and MDC

Every NPC is either an SDC being or an MDC being; the NPC mdc property switches between the two models.

  • SDC beings (mdc 0, the default) have Hit Points underneath (the hp property, stored as rifts_hp) and can also carry an SDC pool on top. SDC works like a buffer: incoming damage drains SDC first, and only the overflow reaches HP. At 0 HP the being falls unconscious; at -10 HP it dies. QCS-generated NPCs start with HP only (SDC 0). To give one an SDC buffer, hand-edit the file and add set_stats("SDC", n); and set_stats("max_SDC", n); after the hp lines; combat honors them automatically.
  • MDC beings (mdc N with N above zero) are tracked as Mega-Damage Capacity directly: one MDC pool, no HP/SDC layering. SDC weapons can never damage an MDC being's MDC pool; only MDC weapons can bring one down.

Worn armor sits in front of either model: damage drains the armor's own MDC or SDC pool first, then overflows to the wearer (SDC buffer, then HP, for SDC beings; the MDC pool for MDC beings).

Where files land

/realms/<you>/area/room/
/realms/<you>/area/npc/

Example log

Below is a short building session using AetherMUD's QCS. Commands are shown as you type them; the following blocks are the QCS replies (and, where noted, the file QCS wrote). The wizard name in the paths is coder; substitute your own. Start from your workroom with the Staff of Creation in inventory.

Room creation

Create a room north of your workroom. QCS writes the new file, wires the forward exit from your current room, and adds the reverse exit back.

create room north practice_room
QCS: wrote and updated /realms/coder/area/room/practice_room.c
QCS: wrote and updated /realms/coder/workroom.c
QCS: room created. Exit 'north' wired to /realms/coder/area/room/practice_room

Walk into the new room, then set short and long. modify here always edits the room you are standing in, so move first.

north
modify here short A practice chamber
QCS: wrote and updated /realms/coder/area/room/practice_room.c
modify here long A bare chamber for learning the builder.
QCS: wrote and updated /realms/coder/area/room/practice_room.c

To link another existing room without creating a new one, use add exit with a full path:

add exit east /realms/coder/area/room/armory
QCS: wrote and updated /realms/coder/area/room/practice_room.c

Inspect the file QCS produced:

about here
/* /realms/coder/area/room/practice_room.c */
/* QCS room by Coder */

#include <std.h>
#include <rooms.h>

inherit ROOM;

void create() {
    ::create();
    set_property("light", 2);
    set_property("indoors", 0);
    set_short("A practice chamber");
    set_long("A bare chamber for learning the builder.");
    add_exit("south", "/realms/coder/workroom");
    add_exit("east", "/realms/coder/area/room/armory");
}

NPC creation

Create an NPC file. The filename becomes the id (set_id(({ "bandit" }))); there is no separate QCS verb for ids or adjectives. Use modify for name, short, long, and the other NPC properties listed above. Hand-edit the file if you need extra ids.

create npc bandit
QCS: wrote and updated /realms/coder/area/npc/bandit.c
QCS: NPC written to /realms/coder/area/npc/bandit.c
modify bandit name bandit
modify bandit short a lean bandit
modify bandit long A lean bandit in scavenged armor watches the exits.
modify bandit race human
modify bandit level 5
QCS: wrote and updated /realms/coder/area/npc/bandit.c

Check the generated NPC:

about bandit
/* /realms/coder/area/npc/bandit.c */
/* QCS NPC by Coder */

#include <std.h>

inherit "/std/rifts_npc";

void create() {
    ::create();
    set_name("bandit");
    set_npc_name("Bandit");
    set_id( ({ "bandit" }) );
    set_short("a lean bandit");
    set_long("A lean bandit in scavenged armor watches the exits.");
    set_level(5);
    set_max_hp(20);
    set_hp(20);
    set_stats("rifts_hp", 20);
    set_aggressive(0);
    set_moving(0);
    set_gender("neuter");
    set_race("human");
    setenv("rifts_occ", "mercenary");
    set_alignment(0);
    set_body_type("human");
    set_property("mdc_creature", 0);
    set_stats("MDC", 0);
    set_stats("max_MDC", 0);
    set_default_response("The creature watches silently.");
}

Note the durability lines: this bandit is an SDC being. Its 20 Hit Points (set_stats("rifts_hp", 20)) are what drops it, and it has no SDC buffer yet, so hits reach HP directly. To layer 30 SDC on top of those Hit Points, hand-edit the file and add set_stats("SDC", 30); and set_stats("max_SDC", 30); after the hp lines; attackers then chew through the 30 SDC before touching HP. The three mdc lines stay zero for an SDC being. Running modify bandit mdc 90 would instead make it an MDC creature with a single 90-MDC pool and no HP/SDC layering at all.

damage writes three properties (damage_num, damage_sides, damage_string); mdc 1 also clears the SDC weapon flag. That is why one typed command can print more than one wrote and updated line.

about vibro_knife
/* /realms/coder/area/weap/vibro_knife.c */
/* QCS weapon by Coder */

#include <std.h>

inherit "/std/weapon";

void create() {
    ::create();
    set_name("vibro knife");
    set_id( ({ "vibro_knife", "weapon" }) );
    set_short("a vibro-knife");
    set_long("A short blade that hums with ultrasonic vibration.");
    set_mass(50);
    set_value(100);
    set_wc(5);
    set_type("blade");
    set_property("sdc_weapon", 0);
    set_property("mdc_weapon", 1);
    set_property("damage_num", 2);
    set_property("damage_sides", 6);
    set_property("damage_string", "2d6");
}

Adding it to a room / cloning it in

Permanent spawn: from inside the room (here, practice_room), add the NPC to a new reset(). QCS refuses if the room already has a reset(); in that case edit the file by hand.

add bandit to here
QCS: wrote and updated /realms/coder/area/room/practice_room.c

That appends a guarded spawn like:

void reset() {
 ::reset();
 if(!present("bandit", this_object()))
  clone_object("/realms/coder/area/npc/bandit")->move(this_object());
}

Quick test without waiting on reset: clone the weapon into inventory (or clone the NPC via the staff clone verb, which places living objects in the room).

clone vibro_knife
QCS: cloned /realms/coder/area/weap/vibro_knife.c to your inventory.

Tips

  • Use short lowercase filenames with underscores; QCS sanitizes them
  • create room already links exits both ways when you are in your realm
  • Spawn NPCs from room reset() with present() guards (add <npc> to here)
  • Test with clone before promoting content into /domains/
  • Generated files are plain LPC; polish with ed, then update
